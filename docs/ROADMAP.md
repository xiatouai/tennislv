# Roadmap — TennisLV

## Phase 1：H5 评级传播（当前阶段）

**目标：** 通过 H5 页面获客，用户完成评级并分享传播。

**时间：** 2026 年 5 月（进行中）

**交付：**

| 模块 | 内容 | 状态 |
|------|------|------|
| NTRP 风格问卷 | 8 题，对应 NTRP 能力维度 | ✅ |
| 评分算法 | 加权评分，输出 1.0-7.0 等级 | ✅ |
| 评级结果页 | 等级、名称、典型特征、维度评分 | ✅ |
| 分层评级 | 问卷预估 / 视频增强 / 球友认证 / 教练认证 | ✅ |
| 视频上传 | 可选上传，mock 流程 | ✅ |
| 视频增强评级 | 视频暂存后独立触发分析 | ✅ |
| 分享评级卡 | 含 NTRP 风格等级、特征、QR 码 | ✅ |
| 分享海报 | 可截图保存 | ✅ |
| 球友认证 | 投票（偏低/差不多/偏高）+ "打过/没打过" | ✅ |
| URL 分享 | hash-based 分享链接 | ✅ |

**不包含：**
- 微信登录 / 账户系统
- 真实 AI 模型调用
- 持久化存储（用内存 Map mock）
- 小程序端

---

## Phase 2：小程序账户与等级档案

**目标：** 用户能在小程序中登录、保存评级、查看等级档案。

**预估时间：** 4-6 周

**后端：**

| 模块 | 内容 |
|------|------|
| 微信登录 | wx.login → openid/session_key，生成 JWT |
| 用户资料 CRUD | 昵称、头像、城市、常打球场 |
| 评级持久化 | 评级结果写入数据库，绑定 userId |
| 评级历史 API | 按时间排序的评级记录 |
| 认证记录 API | 我收到的球友认证列表 |
| 分享码 API | 生成小程序码 + 分享链接 |

**小程序端：**

| 页面 | 内容 |
|------|------|
| 首页 / 评级入口 | 评级问卷组件 |
| 评级结果页 | 同 H5，适配小程序 |
| 我的页面 | 等级档案、评级历史、设置 |
| 分享卡片 | 小程序原生分享 + Canvas 海报 |

**数据：**
- 引入 PostgreSQL + Prisma
- User、Rating、PeerVerification 表
- 迁移 H5 内存数据到数据库

---

## Phase 3：同城球友

**目标：** 用户能找到同城、同水平的球友并建立联系。

**预估时间：** 6-8 周

**后端：**

| 模块 | 内容 |
|------|------|
| 城市 / 球场 | 城市列表、球场 POI 数据 |
| 用户搜索 | 按城市 + 等级范围筛选 |
| 球友推荐 | 基于等级匹配 + 活跃度排序 |
| 好友系统 | 申请 / 确认 / 拒绝 / 解除 |
| 好友列表 | 互相确认后的球友关系 |

**小程序端：**

| 页面 | 内容 |
|------|------|
| 球友 Tab | 推荐球友列表 + 搜索 |
| 球友详情 | 等级标识、常打球场、打球频率 |
| 申请页面 | 发送 / 查看申请 |
| 我的球友 | 已确认的球友列表 |

**数据：**
- City、Court、Friendship 表
- User 增加 cityId、homeCourts、playFrequency

---

## Phase 4：同城约球

**目标：** 用户可以发布约球局，其他球友报名参与。

**预估时间：** 6-8 周

**后端：**

| 模块 | 内容 |
|------|------|
| 球局 CRUD | 时间、球场、人数、等级要求 |
| 球局搜索 | 按城市 + 时间 + 等级筛选 |
| 报名系统 | 申请 / 确认 / 拒绝 / 取消 |
| 球局分享 | 生成分享链接/小程序码 |
| 赛后评价 | 球友互评（水平 + 态度） |

**小程序端：**

| 页面 | 内容 |
|------|------|
| 约球 Tab | 球局列表（地图/列表视图） |
| 球局详情 | 时间、地点、已报名球友 |
| 发布球局 | 表单：时间、球场、人数、等级 |
| 我的球局 | 我发起 / 我报名的球局 |

**数据：**
- Match、MatchRegistration、MatchReview 表

---

## Phase 5：评级校准

**目标：** 多源反馈持续提升评级准确度和置信度。

**预估时间：** 4-6 周

**后端：**

| 模块 | 内容 |
|------|------|
| 球友认证加权 | 打过球 > 没打过，多次认证综合计算 |
| 教练认证 | 教练角色 + 认证权限 |
| 对战反馈 | 赛后评价纳入评级模型 |
| 动态置信度 | 多源反馈综合调整置信度分数 |
| 等级历史图表 | 等级随时间变化的可视化数据 |

**小程序端：**

| 页面 | 内容 |
|------|------|
| 教练认证入口 | 教练专属功能 |
| 等级变化图表 | 评级历史趋势 |
| 认证来源明细 | 各来源对评级的贡献 |

**数据：**
- Coach 角色、Certification 表
- RatingAdjustment 事件日志
- 评级算法升级支持多源输入

---

## 长期愿景（Phase 6+）

- **赛事系统**：用户自发组织比赛，系统记录战绩
- **等级排行榜**：按城市/俱乐部排名
- **训练计划**：基于等级和弱项生成个性化训练建议
- **视频分析**：接入真实多模态模型进行动作分析
- **俱乐部系统**：俱乐部创建、成员管理、内部排名
- **开放平台**：场馆/教练入驻，提供场地预订和教练预约
