# Social & Matching — TennisLV

## 概述

TennisLV 的社交和匹配系统旨在连接同城同水平的业余网球爱好者，降低约球门槛。

**当前状态：** Phase 1 H5 阶段不包含社交功能。本文档为 Phase 3-5 的设计参考。

## 匹配策略

### 等级匹配

根据用户的 NTRP 风格等级进行匹配。

**匹配规则：**

| 场景 | 等级范围 | 说明 |
|------|---------|------|
| 球友推荐 | ±0.5 | 例如 3.0 推荐 2.5-3.5 |
| 约球局 | ±0.5 或自定义 | 发起人可设置 min/max |
| 教练认证 | 不限 | 教练可评定任意等级 |

**匹配算法权重：**

```
匹配分 = 等级接近度 × 0.5 + 同城 × 0.3 + 同球场 × 0.2
```

- 等级接近度 = 1 - |用户等级 - 目标等级| / 3.0
- 同城 = 1 if 同城 else 0
- 同球场 = 1 if 共享常打球场 else 0

### 推荐排序

```
球友推荐排序：
1. 等级匹配度（降序）
2. 同球场优先
3. 活跃度（最近登录时间）
4. 认证数（球友认证数多的优先）
```

## 球友系统（Phase 3）

### 用户流程

```
[球友 Tab]
  → 查看推荐球友（基于等级 + 城市）
  → 筛选：城市 / 等级范围 / 常打球场
  → 查看球友详情（等级标识、常打球场、打球频率）
  → 发送加球友申请
  → 对方收到申请通知
  → 确认 → 互为球友
```

### 球友详情展示

| 字段 | 说明 |
|------|------|
| 头像 + 昵称 | 基础信息 |
| NTRP 风格等级 | 等级 + 名称，如 "3.0 中级入门" |
| 评级类型标识 | 问卷预估 / 视频增强 / 球友认证 / 教练认证 |
| 常打球场 | 最多 3 个 |
| 打球频率 | 几乎每天 / 每周 2-3 次 / 每周 1 次 / 偶尔 |
| 球龄 | 5 年以上 / 2-5 年 / 1-2 年 / 不到 1 年 |
| 风格标签 | 底线型、进攻型、上网型 等 |
| 球友数 | 已确认的球友数量 |
| 认证数 | 收到的球友认证数量 |

### 球友上限

- 初始球友上限：50 人
- 可通过认证或活动参与提升

## 约球系统（Phase 4）

### 球局模型

```
球局状态流转：
open → full（满员）→ completed（结束）
  ↓
cancelled（取消）
```

### 发布球局

**必填项：**

| 字段 | 说明 |
|------|------|
| 标题 | 如 "周六上午 3.0 约球" |
| 日期 | YYYY-MM-DD |
| 时间 | HH:mm - HH:mm |
| 球场 | 从球场列表选择 |
| 人数 | 总名额（含发起人） |
| 等级要求 | minLevel - maxLevel |

**选填项：**

| 字段 | 说明 |
|------|------|
| 费用 | 人均场地费（元） |
| 备注 | 自由文本 |

### 报名流程

```
[查看球局详情]
  → 检查等级要求（用户等级是否在范围内）
  → 点击"报名"
  → 发起人收到通知
  → 发起人确认 / 拒绝
  → 报名人收到结果通知
```

### 球局列表

**筛选条件：**

| 筛选项 | 说明 |
|------|------|
| 城市 | 默认当前城市 |
| 日期 | 今天 / 明天 / 本周末 / 自定义 |
| 等级 | 适合我的（默认） / 全部 |
| 距离 | 离我最近 / 不限 |

**列表展示：**

| 字段 | 说明 |
|------|------|
| 标题 | 球局标题 |
| 时间 | 日期 + 时间 |
| 球场 | 球场名称 |
| 等级要求 | 如 "2.5-3.5" |
| 名额 | 剩余/总名额 |
| 费用 | 人均费用 |
| 发起人 | 等级 + 昵称 |

### 球局分享

- 生成球局小程序码
- 分享到微信群
- 分享文案："本周六下午 3.0-3.5 约球，缺 2 人，来吗？"

## 赛后评价（Phase 5）

### 评价维度

| 维度 | 说明 | 分值 |
|------|------|-----|
| 技术水平 | 对方的技术水平 | 1-5 |
| 球品 | 比赛态度、礼仪 | 1-5 |
| 评语 | 文字评价 | 可选 |

### 评价对评级的影响

- 技术水平评价纳入"球友认证"计算
- ≥3 次赛后评价可触发"球友认证"升级
- 评价数据用于调整推荐排序

## 球场数据

### 球场信息结构

```typescript
interface Court {
  id: string;
  name: string;          // "朝阳公园网球中心"
  city: string;          // "北京"
  cityCode: string;      // "110000"
  district: string;      // "朝阳区"
  address: string;       // 详细地址
  latitude: number;
  longitude: number;
  courtCount: number;    // 场地数
  hasLights: boolean;    // 灯光
  hasRoof: boolean;      // 顶棚
  surface: string;       // 硬地/红土/草地
}
```

**球场数据来源（Phase 3）：**
1. 预置主流城市球场数据
2. 用户自定义添加球场
3. 地图 POI 搜索

## 通知系统（Phase 3+）

### 通知类型

| 类型 | 触发条件 |
|------|---------|
| 球友申请 | 收到加球友申请 |
| 申请通过 | 球友申请被通过 |
| 球局报名 | 有人报名我的球局 |
| 报名确认 | 报名被确认/拒绝 |
| 球局提醒 | 球局开始前 2 小时 |
| 赛后评价 | 收到球友评价 |
| 认证提醒 | 球友邀请认证 |

### 通知渠道

| Phase | 渠道 |
|-------|------|
| Phase 1 | 无（H5 无账户） |
| Phase 3 | 小程序订阅消息 |
| Phase 4 | 小程序订阅消息 + 服务号模板消息 |

## 隐私与安全

- 用户可设置等级档案可见性（公开 / 球友可见 / 仅自己）
- 球友申请需要双方确认
- 约球局报名需发起人确认
- 评价内容仅双方可见
- 城市信息可隐藏
- 手机号不公开显示
